    CaptureConstraint

    A capture constraint is a pair of variables $(b,f)$, where the former appears as a bound variable (hence the name $b$) and in its scope, the latter appears free (hence the name $f$). It is called a capture constraint because the variable $f$ cannot be replaced by any expression containing $b$ free, or variable capture would occur. (The reader may recall that the standard definition of variable capture is that a variable that appears free in an expression becomes bound when that expression is placed in a new context, typically through substitution.)

    This class will store the $b$ and $f$ internally, but note that either of them might be a metavariable. Such metavariables are interpreted as meaning "we do not yet know the value of this variable, but it will be determined later, and replaced here when we determine it." Consequently, we might have a capture constraint whose meaning is still uncertain, until the metavariables in it have been instantiated. See the complete() function below for more details.

    Constructor

    new CaptureConstraint(bound, free)

    Construct a capture constraint $(b,f)$, as defined above, by specifying the bound and free variables $b$ and $f$, in that order, as parameters. Although both should be Symbols, the latter may be a metavariable that is later replaced by an entire expression.

    Parameters

    • bound Symbol

      the bound variable $b$ in the definition of capture constraint at the top of this page

    • free Symbol

      the free variable $f$ in the definition of capture constraint at the top of this page

    complete() → {boolean}

    A capture constraint is complete if neither its bound nor free variables is a metavariable. If either is a metavariable, its eventual value has not yet been determined (or not yet communicated to this object) and thus we can't judge everything about it that we might like to (such as satisfied() or violated()). This function lets us know whethe we have enough information to evaluate such predicates.

    equals(other) → {boolean}

    Recall that the bound and free members of this class, provided at construction time, are instances of the MathConcept class. (They begin as Symbols, but may be substituted later.) One instance of this class equals another if their corresponding bound and free members are equal, as per the equality checking routine in the MathConcept class.

    satisfied() → {boolean|undefined}

    A constraint is satisfied if its bound variable does not occur free in its free expression. Although this suggests that this function should return a boolean, it actually may return undefined if this object is not complete(), because we don't have enough information to answer the question.

    violated() → {boolean}

    A constraint is violated if it is complete and not satisfied. That is, a constraint cannot be violated unless we have enough information to say whether it is satisfied (hence the check for whether it is complete()), but once it is complete, then violated means the same thing as not satisfied().

    Unlike satisfied(), this function will never return undefined. The constraint is not yet violated if we do not yet have non-metavariable values for both the bound and free members.

    • satisfied()

    • boolean

      whether this constraint is (known to be) violated
